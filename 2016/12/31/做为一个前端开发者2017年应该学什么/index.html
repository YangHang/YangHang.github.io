现在前端的环境发展的特别快，我们把大量的时间花费在了试用和争论各种新发明的轮子上。我并不是说我们不应该这么做，但是我觉得我们更应该慢下来，看那些没有太大变化的东西，这会极大的提高我们的工作质量和价值，并且有助于我们理解新的工具。

这篇文章包含了我自己的一些经验和对于新的一年的期望，希望大家可以像我一样发表自己的建议。

学习怎么写可读性高的代码

其实我们大部分的时间不是在写新代码而是在修改已存在的代码，这就意味着，我们读代码比写代码更加频繁，所以你要为了下一个看你代码的程序员优化自己的代码，而不是为了编译器。我建议读一下这三本书——按照书名长短排序

  • The Art of Readable Code by Dustin Boswell
  • Clean Code: A Handbook of Agile Software Craftsmanship by Robert C. Martin
  • Code Complete: A Practical Handbook of Software Construction by Steve McConnell

深入学习JavaScript

现在基本上每一周都会出现一个新的更好的JavaScript框架，人们一般都把大部分时间花费在了学习新的框架上面而不是学习语言本身。如果你使用了一个框架但是不明白它是怎么工作的，你应该停下来，先学习JavaScript，直到你知道了它的工作原理

  • Kyle Simpson的You Don’t Know JavaScript系列丛书是一个学习JavaScript的好的开始,并且可以在网上免费阅读
  • Eric Elliott有一个大的列表JavaScript topics to learn in 2017.
  • Henrique Alves也有一个列表关于在你使用Rect之前需要了解的东西(实际上适用于任何框架)
  • JavaScript Developers: Watch Your Language by Mike Pennisi——了解TC-39为新的ECMAScript特性做的一些处理

学习函数式编程

我们已经期待js有class很多年了，现在js终于有了，但是我们却不想在用了，函数式编程才是我们想要的，我们甚至在写HTML的时候都在用函数(JSX)

  • Functional-Light JavaScript by Kyle Simpson
  • Professor Frisby的Mostly adequate guide to functional programming ebook,还有他的免费课程

学习基本的设计知识

作为一个前端开发者，我们比其他任何人甚至于设计师都更接近用户，如果设计师需要自己验证你写的界面上的每一个像素，那么你就做错了什么东西。

  • Design for Hackers: 一本书和一个免费课程
  • Design for Non-Designers Tracy Osborn主讲
  • Design of Web Applications by Nathan Barry.
  • On Web Typography by Jason Santa Maria.
  • The Inmates Are Running the Asylum: Why High Tech Products Drive Us Crazy and How to Restore the Sanity by Alan Cooper.
  • 一些关于UI动画的文章 How to Use Animation to Improve UX, Transitional Interfaces.

学习怎么和人一起工作

我们中的一些人之所以成为程序员，是因为比起人来，我们更喜欢和电脑沟通-不幸的是，这是不正确的

我们基本上不会独立工作，我们必须和其他的开发者，设计师，经理甚至是用户来沟通，虽然这挺难的，但是如果你真的想明白，你在做什么和你为什么做这个，你就必须这么做,因为这是我们做的东西产生价值的所在。

  • Soft Skills: The software developer’s life manual by John Sonmez.
  • The Clean Coder: A Code of Conduct for Professional Programmers by Robert C. Martin.
  • Start with No: The Negotiating Tools that the Pros Don’t Want You to Know by Jim Camp.

学习怎么给人写东西

我们在和大学同学还有其他人沟通的时候，很大一部分是通过文字，任务说明，评论，代码注释，git commit，聊天信息，邮件，微博，博客等等
想象一下，人们花费了多少时间在阅读和理解上。如果你可以通过写的更新清晰易懂，你可以替他们节省大量的时间，世界将会变得更加美好。

  • On Writing Well: The Classic Guide to Writing Nonfiction by William Zinsser
  • The Elements of Style by William Strunk and E. B. White
  • Orwell’s rules on writing
  • In Russian: awesome Glavred course

学习以前的计算机学科的经验智慧

现在前端的开发者不在为下拉菜单做动画了，现在比以前更加复杂了，人们对于臭名昭著”Javascript疲倦”一部分源自于我们必须解决任务的复杂性的增长。
这就意味着，是时候学习一些非前端开发者经过几十年积累起来的经验教训了，所以这里是需要大家的建议做多的地方

  • Learn To Think Like A Computer Scientist course at Coursera.
  • The five programming books that meant most to me by David Heinemeier Hansson
